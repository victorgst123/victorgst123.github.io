Key concepts: A pipe is a unidirectional byte stream managed by the kernel. pipe() returns two file descriptors — fd[0] for reading and fd[1] for writing. Data is stored in a kernel buffer (default 64 KB on Linux) and follows FIFO order.

Blocking behavior: If the buffer is full, write() blocks the writer until space is available. If the buffer is empty, read() blocks the reader until data arrives. When all writers close fd[1], the reader gets EOF. When all readers close fd[0], the writer receives SIGPIPE.
