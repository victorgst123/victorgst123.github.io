Multi-byte I/O: Each write() call pushes an entire string into the buffer in one syscall — not byte by byte. Each read(fd, buf, n) pulls up to n bytes in one syscall, returning however many are currently available.

Partial writes: If the buffer doesn't have enough room for the full string, write() writes as many bytes as it can and returns that count. The program must loop to send the rest. Try it — type a long string when the buffer is nearly full.

Blocking behavior: Buffer full → write() blocks until space is available. Buffer empty → read() blocks until data arrives. All writers close fd[1] → reader gets EOF (return 0). All readers close fd[0] → writer receives SIGPIPE.
